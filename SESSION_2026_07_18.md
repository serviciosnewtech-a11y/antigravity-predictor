### 2. Retrain Pipeline — Full Bug Log & Fixes

The pipeline (`retrain_all.sh`) crashed **six times** before completing. Every error and fix is recorded here in order.

#### Crash 1 — `write_parquet` missing from `lgbm_poc.io`
- **File:** `src/lgbm_poc/io.py`
- **Error:** `ImportError: cannot import name 'write_parquet' from 'lgbm_poc.io'`
- **Cause:** The original `lgbm_poc/io.py` was a shim (`from io import *`) with no custom functions
- **Fix:** Added `write_parquet`, `read_parquet`, `ensure_parent` directly to `lgbm_poc/io.py`

#### Crash 2 — `lgbm_poc.evaluate` and `lgbm_poc.train` didn't exist
- **Files:** `src/lgbm_poc/evaluate.py`, `src/lgbm_poc/train.py`
- **Error:** `ModuleNotFoundError` for both
- **Fix:** Created both as shims (`from evaluate import *`, `from train import *`) — **this was wrong** and led to Crash 4

#### Crash 3 — `pd.NA` in `_lower_tf_features` breaks pandas rolling
- **File:** `src/prepare_full_dataset.py`
- **Error:** `pandas.errors.DataError: No numeric types to aggregate`
- **Cause:** `.replace(0, pd.NA)` produces `object` dtype; pandas rolling requires `float64`
- **Fix:** Replaced all `pd.NA` with `np.nan` in `_lower_tf_features`, `_tf_features`, `_ctx_features`

#### Crash 4 — stdlib `io` collision / `ensure_parent` not importable
- **File:** `src/train.py` (the shim's target)
- **Error:** `ImportError: cannot import name 'ensure_parent' from 'io' (/usr/.../io.py)`
- **Root cause:** `src/io.py` sat on `sys.path` and **also** shadowed the stdlib `io` module in some contexts but not others. The shim-layer pattern (`lgbm_poc/train.py` → `from train import *` → `src/train.py` → `from io import ensure_parent`) resolved `io` to the stdlib, not `src/io.py`
- **Fix:** This triggered the architectural rethink (see Section 3 below)

### 3. Architectural Refactor — `lgbm_poc` Package (Opus-level fix)

**Root cause of the entire crash chain:** The `lgbm_poc/` package was a thin shim layer — every file did `from <module> import *` to re-export from `src/*.py`. This pattern is fragile because:
- Shims resolve imports at import-time, inheriting whatever `sys.path` state exists
- `src/io.py` on `sys.path` shadowed Python's stdlib `io` — any `import io` by pandas/sklearn/lightgbm could resolve wrong
- Relative imports (`from .baseline import X`) inside `src/*.py` crash when those files are imported standalone via the shim
- Two parallel implementations meant fixes in one layer were silently ignored by the other

**Fix (Opus agent, complete rewrite):**
- Deleted the shim pattern entirely from every `lgbm_poc/*.py` file
- Moved real implementations into `lgbm_poc/` with clean `from lgbm_poc.X import Y` imports throughout
- Moved the 8 orphaned `src/*.py` duplicates (`io`, `baseline`, `features`, `labels`, `futures`, `dataset`, `evaluate`, `train`) into `src/_legacy/` — off the import path
- `src/io.py` is gone — stdlib `io` is safe

**Clean import chain (post-fix):**
```
train_lightgbm.py
  → import _bootstrap                    # adds src/ to sys.path[0]
  → from lgbm_poc.baseline  import ...  # real impl, no deps
  → from lgbm_poc.io        import ...  # real impl, no stdlib collision
  → from lgbm_poc.dataset   import ...  # real impl, 126 FEATURE_COLUMNS
  → from lgbm_poc.evaluate  import ...  # real impl, uses lgbm_poc.dataset
  → from lgbm_poc.train     import ...  # real impl, uses lgbm_poc.dataset + lgbm_poc.io
```

**Files rewritten in `src/lgbm_poc/`:**
- `baseline.py` — real `BaselineSpec`, `BASELINE`, `baseline_dict`
- `features.py` — real feature engineering, `from lgbm_poc.baseline import BASELINE`
- `labels.py` — real labeling (long + short), same clean import
- `futures.py` — real futures frame builder
- `resample_ohlcv.py` — real OHLCV resampler
- `dataset.py` — real `FEATURE_COLUMNS` (126) + `select_xy`
- `io.py` — real `ensure_parent`, `read_parquet`, `write_parquet`
- `evaluate.py` — real `evaluate_model`, `save_metrics`
- `train.py` — real `TrainConfig`, `train_binary_model`, `save_model`
- `__init__.py` — package docstring only
